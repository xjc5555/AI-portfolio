## 二、调用的外部 API

### 1. 搜索类 API（在 `phone_search.py` 中）

#### Serper（优先）

- **地址**：`https://google.serper.dev/search`
- **方法**：POST，JSON body
- **环境变量**：`SERPER_API_KEY`
- **请求示例**：
  - `q`: 查询串（商品名 + 二手 + site:zhuanzhuan.com 等）
  - `num`: 结果数量
- **响应**：`organic[]`，每项含 `title`、`link`/`url`、`snippet`/`description`

#### Google Custom Search（次选）

- **地址**：`https://www.googleapis.com/customsearch/v1`
- **方法**：GET，Query 参数
- **环境变量**：`GOOGLE_API_KEY`、`GOOGLE_CX`（自定义搜索引擎 ID）
- **参数**：`key`、`cx`、`q`、`num`
- **响应**：`items[]`，每项含 `title`、`link`、`snippet`

#### DuckDuckGo（无 key，兜底）

- **库**：`duckduckgo_search`，使用 `DDGS().text(query, max_results=...)`
- **无环境变量**，直接调用
- **用途**：Serper、Google 都未配置或失败时使用

#### Bing（预留）

- **环境变量**：`BING_API_KEY` 在 `phone_search.py` 中已读取，但当前代码未实现调用逻辑。

---

### 2. AI 大模型 API（在 `app.py` 中）

调用顺序：**豆包 → 魔搭 → 阿里百炼**，任一成功即返回，不再调用后面的。

#### 豆包（火山引擎）

- **地址**：`https://ark.cn-beijing.volces.com/api/v3/chat/completions`
- **方法**：POST，JSON
- **环境变量**：`DOUBAO_KEY`
- **请求体**：
  - `model`: `doubao-pro-32k`
  - `messages`: 对话列表（含 system/user/assistant）
  - `max_tokens`: 500 或 1000（分析接口用 1000）
  - `temperature`: 0.7
- **响应**：OpenAI 兼容格式，取 `choices[0].message.content`

#### 魔搭（ModelScope）

- **地址**：`https://api-inference.modelscope.cn/v1/chat/completions`
- **方法**：POST，JSON
- **环境变量**：`MODELSCOPE_KEY`
- **请求体**：
  - `model`: `Qwen/Qwen2.5-72B-Instruct`
  - `messages`、`max_tokens`、`temperature` 同上
- **响应**：同上，取 `choices[0].message.content`

#### 阿里百炼（DashScope 兼容）

- **地址**：`https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
- **方法**：POST，JSON
- **环境变量**：`BAILIAN_KEY`
- **请求体**：
  - `model`: `qwen-turbo`
  - `messages`、`max_tokens`、`temperature` 同上
- **响应**：同上

所有 AI 请求均带 `Authorization: Bearer <KEY>`，超时 30 秒；失败时打日志并尝试下一个引擎，全部失败则返回固定文案「AI服务暂时不可用，请稍后再试。」

## 三、后端实现要点

### 1. 路由与职责（app.py）

| 路由 | 方法 | 功能简述 |
|------|------|----------|
| `/` | GET | 返回 `index.html`，异常时返回简单 HTML 字符串 |
| `/styles.css` | GET | 返回 `styles.css` |
| `/app.js` | GET | 返回 `app.js` |
| `/mockData.js` | GET | 返回 `mockData.js` |
| `/api/search` | GET/POST | 根据 `keyword` 搜索二手商品，真实搜索或 mock |
| `/api/chat` | POST | 单轮对话：`message` + `history` → AI 回复 |
| `/api/conversation/start` | POST | 多轮对话：按 `user_id` 初始化 `conversation_history[user_id]=[]`，返回欢迎语 |
| `/api/conversation/message` | POST | 多轮对话：追加 user 消息，调用 AI，再追加 assistant 消息并返回 |
| `/api/analyze` | POST | 搜索 + 价格统计 + AI 分析，返回完整分析结果 |

### 2. 搜索链路（真实 vs 模拟）

- **是否真实搜索**：通过 `try: from phone_search import search_second_hand as real_search, get_platform_links` 判断，成功则 `HAS_REAL_SEARCH=True`。
- **真实路径**：`/api/search`、`/api/analyze` 中调用 `real_search(keyword, max_results=15)` 和 `get_platform_links(keyword)`。
- **模拟路径**：`mock_search(keyword)`、`get_mock_platform_links(keyword)`，基于关键词匹配预设价格区间（如 iPhone/MacBook/AirPods 等），按平台生成随机价格与成色。

### 3. 搜索模块内部流程（phone_search.py）

- **主入口**：`search_second_hand(product, max_results=15)`。
- **步骤**：
  1. 调用 `_engine_search_main_platforms(product, max_results)`：
     - 若有 `SERPER_API_KEY`：POST 到 Serper，解析 `organic`。
     - 若无结果且有 `GOOGLE_API_KEY`+`GOOGLE_CX`：GET Google Custom Search，解析 `items`。
     - 若无结果：用 `DDGS().text(...)` DuckDuckGo 搜索。
  2. 查询构造：`{product} 二手 出售 闲鱼 转转 京东 site:zhuanzhuan.com OR site:xianyu.com OR ...`，聚焦主流平台。
  3. 每条结果：`detect_platform(href)` 识别平台、`extract_price(title+body)` 正则提取价格、`extract_condition(...)` 提取成色，并计算 `confidence`、标记 `is_real_data`。
  4. 若引擎均无结果：使用 `get_enhanced_mock_data(product, max_results)`，按商品类型（如 iphone/华为/小米/airpods/macbook 等）设定价格区间和成色折扣，生成增强模拟数据。
- **平台链接**：`get_platform_links(keyword)` 返回闲鱼、转转、京东二手、淘宝、拼多多、58、赶集、拍拍、爱回收的搜索 URL（`quote(keyword)` 编码）。

### 4. 价格与成色解析（phone_search.py）

- **价格**：`extract_price(text)` 使用多条正则，支持如 `¥1234`、`1234元`、`价格：1234`、区间 `¥1000-¥2000`、美元/欧元/英镑等，取到则返回数值（区间取中值）。
- **成色**：`extract_condition(text)` 关键词匹配「99新」「95新」「9成新」「8成新」「7成新以下」等，返回统一标签。

### 5. AI 调用场景（app.py）

- **单轮聊天**：`call_ai_chat(message, history)`，组装 system（赛博淘客助手）+ history + 当前 message，调用 `call_ai_api(messages)`。
- **多轮对话**：`call_ai_conversation(message, conversation_history[user_id])`，system 中约定角色与回复风格（含 emoji、200 字内等），再拼接历史与当前 message。
- **分析报告**：`call_ai_analysis(keyword, stats)`，传入商品名与 `avg_price/min_price/max_price/count`，要求模型输出市场分析、购买建议、注意事项，`max_tokens=1000`。

## 四、前端实现要点

### 1. 请求与缓存（app.js）

- **分析流程**：
  1. `GET /api/search?keyword=xxx` 获取搜索结果。
  2. `POST /api/analyze`，body `{ keyword }`，后端内部会再搜一次并做统计与 AI 分析。
- **前端缓存**：`analysisCache = new Map()`，key 为商品名，value 为 `{ data, timestamp }`；`CACHE_DURATION = 5 * 60 * 1000`（5 分钟）。同一商品在 5 分钟内再次分析直接读缓存并 `displayReport(cached.data)`，不再请求接口。

### 2. 数据处理与展示

- **processSearchData(product, searchData, analyzeData)**：用 `searchData.data` 计算均价、最低、最高、中位数、平台统计（闲鱼/转转/爱回收/拍拍的数量与占比），生成 `priceDistribution`、`trendData`（30 点模拟趋势），并合并 `analyzeData.data.aiAnalysis` 与 `platformLinks`。
- **generateMockData(product)**：当接口报错时在前端生成一套随机报告数据，保证报告区仍可展示。
- **displayReport(data)**：填充报告区各 DOM（产品名、价格、平台统计、建议、AI 分析、链接列表），再调用 `drawCharts(data)`。

### 3. 图表（Chart.js）

- **价格分布**：`priceDistributionChart`，条形图，数据来自 `data.priceDistribution`，标签为 `data.priceRanges`。
- **平台分布**：`platformDistributionChart`，饼图/环形图，闲鱼/转转/爱回收/拍拍占比。
- **价格趋势**：`priceTrendChart`，折线图，30 个点 `data.trendData`。
- 每次绘制前对已有实例执行 `destroy()`，避免重复创建。

### 4. 对话

- **多轮**：`CONVERSATION_START_API_URL` 初始化会话；`CONVERSATION_API_URL` 发送每条消息，请求体 `{ user_id, message }`，渲染返回的 `response`。
- **历史**：`ChatHistoryManager` 用 `localStorage['chatHistory']` 存最近 100 条；分析历史用 `HistoryManager` 存 `localStorage['history']`，最多 20 条，展示产品名、时间、均价，点击可把商品名填入输入框并触发分析。

### 5. 主题与其它

- **ThemeManager**：`data-theme` 在 `light`/`dark` 间切换，持久化到 `localStorage['theme']`。
- **图片识别**：`recognizeImage()` 为前端模拟，固定 2 秒后展示「iPhone 13 Pro」等假结果，再 `startAnalysisWithImage()` 把「iPhone 13 Pro」填入并调用 `startAnalysis()`。

### 6. 静态与建议数据（mockData.js）

- **presetProducts**：预设商品列表（部分在 index 里以 chip 展示）。
- **searchSuggestions**：按关键词（如 iphone、小米、华为）给出联想列表，用于输入建议。
- **analyzeResult**：示例结构（价格分布、平台、趋势、推荐等），可供前端参考或占位。

## 六、数据流简图

```
用户输入商品名
    → 前端 startAnalysis()
    → GET /api/search?keyword=xxx  →  phone_search.real_search 或 app.mock_search
    → POST /api/analyze { keyword } → 同上搜索 + 价格统计 + call_ai_analysis()
    → 返回 { data: { avgPrice, minPrice, maxPrice, results, platformLinks, aiAnalysis } }
    → 前端 processSearchData() → displayReport() → drawCharts()
```

```
用户点击「智能对话」
    → POST /api/conversation/start { user_id }
    → 用户发消息 → POST /api/conversation/message { user_id, message }
    → 后端 conversation_history[user_id].append(user) → call_ai_conversation() → append(assistant)
    → 返回 { response } → 前端 appendBotMessage()
```

## 七、MCP / Open Claw 接入（赛博淘客 × Open Claw）

将赛博淘客从「被动比价 Web 工具」进化为「24 小时捡漏外挂」时，通过 **MCP (Model Context Protocol)** 把 `phone_search` 能力暴露为 Agent 工具，供 Open Claw 等本地 Agent 以 Tool 方式调用，无需推翻原有 Flask 架构。

### 1. MCP Server：mcp_server.py

- **协议**：FastMCP（`fastmcp` 包），默认 **stdio** 传输，便于 Open Claw 子进程挂载。
- **入口**：`python mcp_server.py` 或 `fastmcp run mcp_server.py`。
- **依赖**：与现有项目一致，需 `phone_search` 可导入（同目录或 PYTHONPATH）。

**暴露的 Tools：**

| 工具名 | 说明 | 限流 |
|--------|------|------|
| `search_second_hand_market(product_name, max_results=10)` | 搜索闲鱼/转转/京东等二手实时价格，返回 Agent 易读的文本摘要 | ✅ |
| `get_market_baseline(product_name)` | 获取均价/最低价/建议捡漏价等基准线，返回 JSON | ✅ |
| `get_platform_links(product_name)` | 获取各平台直达搜索链接，便于用户自行打开比价 | ✅ |

**风控与权限：**

- **Token Bucket 限流**：默认容量 10、每 6 秒补充 1 token，约 10 次/分钟，避免 Serper/DuckDuckGo 频次限制或封 IP。被限流时返回友好提示及建议重试时间。
- **Human-in-the-loop**：仅提供查询(Read)，不提供支付/下单(Write)；所有决策最后一步由用户点击链接完成支付。

### 2. Open Claw 端配置

在 Open Claw 的配置文件（如 `claw-config.json`）中合并以下 `mcpServers` 片段（路径按本机实际修改）：

**Windows 示例：**

```json
{
  "mcpServers": {
    "cyber_taoke": {
      "command": "python",
      "args": ["C:/Users/10761/github-project/AI-Decision3/mcp_server.py"]
    }
  }
}
```

**Linux / macOS 示例：**

```json
{
  "mcpServers": {
    "cyber_taoke": {
      "command": "python",
      "args": ["/path/to/your/AI-Decision3/mcp_server.py"]
    }
  }
}
```

项目根目录下提供 `open-claw-mcp-config.example.json`，可直接复制到 Open Claw 配置中并修改 `args` 为本地绝对路径。

**环境变量**：MCP Server 与 Flask 共用同一进程环境，若需更好搜索效果，在启动 Open Claw 的环境中配置 `SERPER_API_KEY`（或 `GOOGLE_API_KEY` + `GOOGLE_CX`），`phone_search` 会按现有逻辑优先使用 Serper/Google。

### 3. 典型 Agent 工作流（Use Cases）

- **捡漏模式**：用户指令「盯住闲鱼上的 iPad Pro M4，跌破 6000 且描述含箱说全就通知我」。Claw 定时调用 `search_second_hand_market("iPad Pro M4")`，本地模型过滤干扰项，命中规则后通过通知模块 Push 用户；**不自动下单**，仅推送链接。
- **顾问模式**：用户「预算 4000 想买拍照好的手机」。Claw 先通过记忆/搜索得出候选（如 vivo X100 Pro、iPhone 13 Pro Max），再调用 `get_market_baseline` 获取当前二手均价与建议捡漏价，最后用 `get_platform_links` 给出各平台链接，由用户自行打开并支付。

### 4. 本地验证 MCP Server

确保依赖已安装（`pip install -r requirements.txt`），在项目根目录执行：

```bash
python mcp_server.py
```

进程会以 stdio 方式等待 MCP 客户端消息；可用 [MCP Inspector](https://github.com/modelcontextprotocol/inspector) 或支持 MCP 的 IDE 连接并测试上述三个 Tool。

## 八、OpenClaw Skills 接入（无 MCP 时用 Skill 调用赛博淘客）

当 OpenClaw 配置 schema 不支持 `mcp` 时，可通过 **Skills** 让助手用 bash 调用本项目的 CLI，实现「查二手价格」能力。

### 1. 项目内已有内容

- **CLI 脚本**：`cyber_taoke_cli.py`（项目根目录）。用法：`python cyber_taoke_cli.py "商品名" [max_results]`，输出与 MCP 工具 `search_second_hand_market` 一致的文本摘要。
- **Skill 定义**：`skills/cyber-taoke/SKILL.md`，描述何时用、如何用 bash 调用上述脚本。

### 2. 配置 OpenClaw 加载该 Skill

**方式 A：从项目目录加载（推荐）**

在 `~/.openclaw/openclaw.json` 的 `skills` 里增加 `load.extraDirs`，指向本项目的 `skills` 目录（路径按本机修改）：

```json5
{
  "skills": {
    "load": {
      "extraDirs": ["C:/Users/10761/github-project/AI-Decision3/skills"],
      "watch": true
    },
    "entries": {
      "cyber-taoke": { "enabled": true }
    }
  }
}
```

**方式 B：复制到 OpenClaw 工作区**

```powershell
mkdir -p $env:USERPROFILE\.openclaw\workspace\skills\cyber-taoke
copy .\skills\cyber-taoke\SKILL.md $env:USERPROFILE\.openclaw\workspace\skills\cyber-taoke\
```

注意：SKILL.md 里的 `cyber_taoke_cli.py` 路径需为本机实际路径；若复制到工作区，请编辑 SKILL.md 中的脚本路径。

### 3. 生效与验证

- 若 `skills.load.watch` 为 true，保存配置或 SKILL.md 后，下一轮对话即可被识别；否则执行 `openclaw gateway restart` 或让助手「刷新 skills」。
- 在聊天里问：「iPhone 15 二手现在什么价」或「查一下闲鱼上 iPad Pro M4 的行情」，助手应会使用 bash 调用 `cyber_taoke_cli.py` 并汇总结果。

### 4. 与 MCP 的区别

| 方式 | 说明 |
|------|------|
| **MCP** | 网关直接挂 mcp_server.py，Agent 有独立 Tool 调用，当前 OpenClaw 2026.3.7 配置不支持。 |
| **Skill** | 通过 SKILL.md 指导 Agent 用 bash 执行 cyber_taoke_cli.py，输出再由 Agent 整理回复；无需 MCP schema。 |
